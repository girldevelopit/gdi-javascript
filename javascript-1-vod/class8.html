Objects

The Object Data Type

Objects are a JavaScript data type

Like arrays, objects hold a collection of data

JavaScript Objects (w3schools)

Objects vs Arrays

But objects differ from arrays:

  • An object stores data that's specific to itself
  • Objects have a different syntax
  • Objects store data as key:value pairs
  • Objects are not zero-indexed

What is an Object?

One way to understand the concept of objects in JavaScript is to use real world examples

Objects in the Real World

  • A car is an object
  • A car has properties

Car Object

What are some of the properties of this car?

  • 4 wheels

What are actions this car can perform?

It can:

  • Start

Many Things are Objects!

A house is an object with properties: architectural style, number of floors, kitchen, basement (yes or no), price, etc.

Actions that can happen in a house: doors open or close, windows open or close, etc.

It can expire, it can be renewed, etc.

Objects in JavaScript

An object in JavaScript is similar to a real-world object; it has properties (and actions) that are specific to it

Syntax: Object Literal

We can declare objects with a variable name and a pair of curly brackets {} 

// Declare an object literal

let objectName = {} /* An empty object */

// Different syntax from an array:
// let arrayName = []

Object Syntax

The data or content inside of an object are its properties

Each property is separated by a comma

Turning our car into a JavaScript object:

let carObj = {
doors: 4,
color: 'blue',
brand: 'bentley',
year: 2021
}

Key:Value Pairs

The properties of an object are called key:value pairs

brand: 'bentley' is a key-value pair

brand is the key; bentley is the value 

let carObj = {
doors: 4,
color: 'blue',
brand: 'bentley',
year: 2021
}

Objects are Not Zero-Indexed

  • Objects are not like arrays; order is not guaranteed
  • We cannot use index positions to access object data
  • We use the key property to access an object

Accessing an Object

Using the key property, there are two ways to access an object:

  • Dot Notation
  • Bracket Notation

JavaScript Quickieâ€” Dot Notation vs. Bracket Notation

Dot Notation

objectName.key 

let dogObj = {
color: 'brown',

breed: 'bulldog'
}


// Access the color property
let dogColor = dogObj.color

console.log(dogColor)

Bracket Notation

objectName['key'] 

let dogObj = {
color: 'brown',

breed: 'bulldog'
}

// Access the color property
let dogColor2 = dogObj['color']

console.log(dogColor2)

Updating an Object Property

Update the color property

dogObj.color = 'blue'

console.log (dogObj)

// Output:
// {color: 'blue', breed: 'bulldog'}

Adding a New Object Property

Add a new size property

dogObj.size = 'small'

console.log (dogObj)

// Output:
// {color: 'blue', breed: 'bulldog', size: 'small'}

Deleting an Object Key

Use the keyword delete

delete objectName.key 

delete dogObj.size

console.log (dogObj)

// Output:
// {color: 'blue', breed: 'bulldog'}

Creating Object Methods

Objects and Function Properties

In addition to key:value properties, objects can also store their own functions--actions that control or determine their behavior

Object Method

A function inside of an object is called a method

Declaring an Object Method

We can create a method when declaring a object

let toddlerObj = {

age_in_months: 18,

is_cute: true,

new_word: 'no',

// Declaring a function or method
faveWord: function(){
  console.log('Mama')
}
}

Declaring an Object Method

We can also declare a method without using the function keyword 

let toddlerObj = {

age_in_months: 18,

is_cute: true,

new_word: 'no',

// Declaring a method without the function keyword
faveWord(){
  console.log('Mama')
}
}

Declaring an Object Method

Using dot notation, we can add a new method after an object has been declared 

let toddlerObj = {
age_in_months: 18,
is_cute: true,
new_word: 'no',
faveWord(){ 
  console.log('Mama') 
},
}

// Add new method
toddlerObj.throwTantrum = function(){
console.log('Nooooo!')
}

console.log(toddlerObj) // Verify

Calling an Object Method

To call an object method, use dot notation to access the method name and append with ()

objectName.objectMethod() 

let toddlerObj = {
age_in_months: 18,
is_cute: true,
new_word: 'no',
faveWord(){ 
  console.log('Mama') 
},
throwTantrum: function(){
  console.log('Nooooo!')
},
}

toddlerObj.throwTantrum()

Built-in Object Methods

Similar to strings and arrays, objects have built-in or default methods

Object Methods (MDN)

Commonly Used Object Methods

  • Object.keys()
  • Object.values()
  • Object.entries()
  • Object.freeze()
  • Object.seal()
  • Object.fromEntries()

Object Methods (MDN)

Object.keys()

Returns an array of an object's keys

let carObj = {
model: 'BMW',
year: '2020',
color: 'red'
}

console.log( Object.keys(carObj) )

// Output: ['model', 'year', 'color']

Object.keys() - MDN

Object.values()

Returns an array of an object's values

let carObj = {
model: 'BMW',
year: '2020',
color: 'red'
}

console.log( Object.values(carObj) )

// Output: ['BMW', '2020', 'red']

Object.values() - MDN

Object.entries()

Returns arrays of an object's key:value pairs

let carObj = {
model: 'BMW',
year: '2020',
color: 'red'
}

console.log( Object.entries(carObj) )

// Output: 3 arrays

Object.entries() - MDN

Looping Through Objects

for...in Loop

Looping Objects

The for...in loop iterates over the properties of an object

It has the same syntax or setup as the for...of 

for( key in object ){

// code to run

}

JavaScript For In (w3schools)

for...in Loop

  • for...in iterates and returns an object's keys
  • Each key is used to access its respective value 
let carObj = {
model: 'BMW',
year: '2020',
color: 'red'
}

for( key in carObj ){

console.log(key)
console.log( carObj[key] )

}
// Output: model, year, color
// Output: BMW, 2020, red

for...in VS for...of

  • While the for...in loop can iterate over an array, order is not guaranteed
  • Use the for...of for iterable (countable) data such as arrays and strings 
for( key of carObj ){

console.log( carObj[key] )

}
// Output: Error!
// carObj is not iterable
