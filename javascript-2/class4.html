Objects: this Keyword

Object References

When we add methods to our objects, we can use these methods to work with other data inside our objects

JavaScript has a keyword this that allows an object to reference data within itself

This (MDN)

Object Referencing:
An Example

In our toddlerObj example, the faveWord() method is currently hard-coded to display "Mama"

What if we wanted that method to display the value of new_word instead? 

let toddlerObj = {
  age_in_months: 18,
  is_cute: true,
  new_word: 'no',
  faveWord(){ 
    console.log('Mama') 
  },
}

this

We would use the this keyword 

let toddlerObj = {
  
  age_in_months: 18,
  
  is_cute: true,
  
  new_word: 'no',

  faveWord(){ 
    console.log(this.new_word) 
  },
}

toddlerObj.faveWord() // Should output 'no'

this

this means "inside of this current object"

From our example: this.new_word means:

"the current value of new_word inside of this toddler object"

this

the this keyword allows object methods to be versatile or reusable when values inside the object changes

For example, we can now update new_word to be any value--as our toddler grows and expands her vocabulary! 

let toddlerObj = {
  age_in_months: 18,
  is_cute: true,
  new_word: 'no',
  faveWord(){ 
    console.log(this.new_word) 
  },
}

toddlerObj.new_word = 'doggie'

toddlerObj.faveWord() // Should output 'doggie'

Looping Through Objects

for...in Loop

Looping Objects

The for...in loop iterates over the properties of an object

It has the same syntax or setup as the for...of 

for( key in object ){

  // code to run

}

JavaScript For In (w3schools)

for...in Loop

  • for...in iterates and returns an object's keys
  • Each key is used to access its respective value 
let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

for( key in carObj ){

  console.log(key)
  console.log( carObj[key] )

}
// Output: model, year, color
// Output: BMW, 2020, red

for...in VS for...of

  • While the for...in loop can iterate over an array, order is not guaranteed
  • Use the for...of for iterable (countable) data such as arrays and strings 
for( key of carObj ){

  console.log( carObj[key] )

}
// Output: Error!
// carObj is not iterable

Built-in Object Methods

Similar to strings and arrays, objects have built-in or default methods

Object Methods (MDN)

Commonly Used Object Methods

  • Object.keys()
  • Object.values()
  • Object.entries()
  • Object.freeze()
  • Object.seal()
  • Object.fromEntries()

Object Methods (MDN)

Object.keys()

Returns an array of an object's keys

let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

console.log( Object.keys(carObj) )

// Output: ['model', 'year', 'color']

Object.keys() - MDN

Object.values()

Returns an array of an object's values

let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

console.log( Object.values(carObj) )

// Output: ['BMW', '2020', 'red']

Object.values() - MDN

Object.entries()

Returns arrays of an object's key:value pairs

let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

console.log( Object.entries(carObj) )

// Output: 3 arrays

Object.seal() - MDN

.seal()

With Object.seal(), new properties cannot be added to an object, nor can existing ones be removed; but existing properties can be updated

Object.isSealed() method checks if an object is sealed; it returns a boolean

.freeze()

With Object.freeze(), new properties cannot be added to an object, existing ones cannot be removed nor updated; it is frozen!

Object.isfrozen() method checks if an object is frozen; it returns a boolean

Immutable Objects

A frozen or sealed object cannot be reversed; there is no "unseal" or "unfreeze" method

But using the Object.assign() method, our immutable object can be copied or cloned to a new object that is mutable
