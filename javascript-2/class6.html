Destructuring

Destructuring is an ES6 feature that allows us to extract specific data or values from arrays and objects and assign them to new variables

Destructuring Assignment (MDN)

Array vs Object Destructuring in JavaScript (FreeCodeCamp)

Pre ES6:
Assigning Array Values

If we wanted to create new variables from items in an array, we would have to do so individually

let people = ['layla', 'keiko', 'amina', 'temi']

let mySister = people[0]
let myBFF = people[1]
let myCoworker = people[2]
let myCousin = people[3]

console.log( mySister, myBFF, myCoworker, myCousin)

// Output: layla keiko amina temi

ES6: Destructuring Arrays

With destructuring, we can extract values more efficiently -- with less code

let people = ['layla', 'keiko', 'amina', 'temi']

let [ mySister, myBFF, myCoworker, myCousin ] = people

console.log( mySister, myBFF, myCoworker, myCousin)

// Output: layla keiko amina temi

Destructuring Syntax: Arrays

We are copying items from an array into variables

let [ variable1, variable2.....variableN ] = array

Another variation: Omit the array declaration altogether

let [ mySister, myBFF, myCoworker, myCousin ] = ['layla', 'keiko', 'amina', 'temi']

console.log( mySister, myBFF, myCoworker, myCousin)

// Output: layla keiko amina temi

With destructuring, we can extract specific values and leave others out

let people = ['layla', 'keiko', 'amina', 'temi']

let [ mySister, , , myCousin ] = people

console.log(mySister, myCousin)

// Output: layla temi

Destructuring Objects

Destructuring also works with objects; the syntax is similar to how arrays are destructured

Pre ES6: Assigning Object Properties

Let's look at the old way first

const animalSounds = {
  cat:   'meow',
  dog:   'bark',
  snake: 'sssss'
}

const cat   = animalSounds.cat
const dog   = animalSounds.dog
const snake = animalSounds.snake

console.log( cat, dog, snake )

// Output: meow bark sssss

ES6: Destructuring Objects

With objects, we can unpack or extract its values by referencing its keys

const animalSounds = {
  cat: 'meow',
  dog: 'bark',
  snake: 'sssss'
}

const { cat, dog, snake } = animalSounds

console.log( cat, dog, snake )

// Output: meow bark sssss

Destructuring Syntax: Objects

We are copying property values from an object by referencing its keys

let { key1, key2.....keyN } = object

Destructuring Objects

If we prefer to use new variable names instead of the key names, we can assign them this way:

const animalSounds = {
  cat: 'meow',
  dog: 'bark',
  snake: 'sssss'
}

const { 
  cat: catSound, 
  dog: dogSound, 
  snake: snakeSound 
} = animalSounds

console.log( catSound, dogSound, snakeSound )

// Output: meow bark sssss

Destructuring Order in Objects

Because objects are not zero-indexed, the order in which they are destructured does not matter as long as the correct key is referenced

const { dog, snake, cat } = animalSounds

// Will still return the right values
// Output: bark, sssss, meow

Extracting Specific Values

We can extract specific values, but unlike arrays, we don't need to use blank spaces as placeholders

const animalSounds = {
  cat: 'meow',
  dog: 'bark',
  snake: 'sssss'
}

const { dog } = animalSounds

console.log(dog)

// Output: Bark

Spread Operator

...

We use the Spread operator to copy or combine an array or object into another array or object

The Spread operator syntax is signified by three dots: ...

...spread operator (FreeCodeCamp, YouTube)

ES6 Spread Operator (w3schools)

Spread Operator: Arrays

Example 1

let usCities = [ 'New York City', 'Los Angeles' ]

let bigCities = [ 'London', ...usCities, 'Tokyo', 'Barcelona' ]

console.log( bigCities )

// Output: 

// [ 'London', 'New York City', 'Los Angeles', 'Tokyo', 'Barcelona' ]

Spread Operator: Arrays

Example 2

let usCities = ['New York City', 'Los Angeles']

let bigCities = ['London', 'Tokyo', 'Barcelona']

bigCities = [...bigCities, ...usCities]

console.log(bigCities)

// Output: 

// ['London', 'Tokyo', 'Barcelona', 'New York City', 'Los Angeles']

Spread Operator: Arrays

Example 3

let usCities = ['New York City', 'Los Angeles']

let dupUsCities = [...usCities]

console.log(dupUsCities)

// Output: 

// ['New York City', 'Los Angeles']

Spread or Rest?

If this new ... syntax isn't strange or confusing enough, it is the same exact syntax for another ES6 feature called the Rest operator!

Rest Operator

The Rest operator is often used with functions

It does the opposite of the Spread operator--it packs up 'leftover' values into an array

Rest Operator

function addTwoNumbers( a, b, ...otherNumbers ){

  console.log( a + b )
  
  console.log( ...otherNumbers )

}

addTwoNumbers( 2, 4, 5, 6, 12 )

// Output:
// 6
// 5 6 12

Caveat

Unlike the spread operator variable which can be placed in any order, the rest operator parameter must come last

Asynchronous JavaScript:

Promises

JavaScript is a single-threaded language; meaning it executes code in order and can only do so one code at a time

How JavaScript Works (Video)

Imagine you're at a restaurant

You place your order for a drink BUT you can't order your appetizer until you finish your drink

You can't order your entree until you finish your appetizer

And on and on and on...

It would be a very long and tedious meal!

If that sounds like a nightmare dining experience, imagine the equivalent when running a website or an application

Asynchronous JavaScript

Asynchronous JavaScript is how we get around the single-threaded nature of the language, by using specific features and protocols such as:

  • setTimeout()
  • Promises
  • Fetch API
  • Async, Await

Promises

Promises allow us to write code that can be executed asynchronously or out of order, meaning other code can run synchronously without interruption

Promises are predominantly used with server or API calls--requests that may take some time to complete

Promise States

When we write a promise, our code is in one of three states:

  • Pending: Initial state of the operation
  • Fulfilled: The operation was successful
  • Rejected: The operation failed

When a promise is fulfilled, we get some value or data back; when it is rejected, we get an error message--the reason why

Basic Promise Syntax

const myFirstPromise = new Promise()

// All the FUN happens in between the parentheses!

Promise Syntax

Let's create a simple promise to see its full syntax

The scenario:

I promise to bake muffins if you find blueberries at the supermarket; if you don't find blueberries, buy muffins at the bakery instead

const muffinPromise = new Promise()

Full Promise Syntax

const muffinPromise = new Promise( ( resolve, reject ) => {

  let blueberriesInStock = true;

  if( blueberriesInStock ){
    resolve()
  } else {
    reject()
  }

})

muffinPromise.then( () => console.log(`We're baking muffins!`))

muffinPromise.catch( () => console.log(`Boo. Buy muffins instead.`))

Syntax Breakdown

A promise takes two callback functions:

  • resolve: runs when the request is successful
  • reject: runs when the request fails
const muffinPromise = new Promise( ( resolve, reject ) => {

  // if the operation is successful, run resolve()
  // else run reject()
})

muffinPromise.then( () => console.log(`We're baking muffins!`))

muffinPromise.catch( () => console.log(`Boo. Buy muffins instead.`))

Syntax Breakdown

Both callbacks are triggers for two respective promise methods:

  • .then(): code to run when the request is successful
  • .catch(): code to run or error message to display if request fails

Fetch API

The Fetch API is a protocol or method that we use to make requests to external servers

Understanding how promises work is key when or if we have to use the Fetch API--why?

Because a Fetch request always returns a Promise

JavaScript Fetch API Explained by Examples

Basic Fetch Syntax

The fetch method takes in a string url value

The url is the API destination; where the data lives

let variableName = fetch( url )

Fetch Demo

Let's make a fetch request to a public API called dogAPI that returns a random dog image

let getDogImage = fetch( 'https://dog.ceo/api/breeds/image/random' )

console.log( getDogImage )

What happens when we log getDogImage to the console?

We get a successful, or fulfilled or resolved Promise that's pending--this means we have data waiting to be used!

Enter the Promise Handlers!

Remember when we used .then() and .catch() in our muffinPromise example?

We'll use these two methods to unpack our resolved getDogImage promise

let getDogImage = fetch('https://dog.ceo/api/breeds/image/random')
  .then( (res)  => res.json())
  .then( (data) => console.log(data))
  .catch( (err) => console.log(err))

Response Data

We get back an object with data we can now use!
