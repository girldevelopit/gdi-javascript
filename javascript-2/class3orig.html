Arrays Iterators (contd)

Array Iteration Methods

Array iterators loop or iterate through an array and perform an action on each item in the array

Array iterators require a second function to be passed as a parameter

This second function contains the action to be applied to each element

JavaScript Array Iteration (w3schools)

Array Iteration Syntax

// An array

let arrayName = [ item, item, item]

// An array iterator

arrayName.arrayIterationMethod( (a parameter) {

  // Code to apply to each element

} )

Array Iteration Syntax

For example, the .find() method returns the first instance of an element that passes a test condition 

let asianCapitals = ['Seoul', 'Dhaka', 'Manilla', 'Bangkok']

// Find 'Manilla' in the array; return it if present

let result = asianCapitals.find(function(city){
  if(city === 'Manilla'){
    return city;
  }
})

console.log(result) // 'Manilla'

Syntax Breakdown

The .find() array method in its simplest form--waiting for a parameter to be placed in between the () 

// The array
let asianCapitals = ['Seoul', 'Dhaka', 'Manilla', 'Bangkok']

// The .find() array method
let result = asianCapitals.find()

// The output we want: 'Manilla'
// console.log(result)

Syntax Breakdown

The parameter is a second function that will specifically check for the city 'Manilla'

When used by the .find() method, the function will be applied to each item in the array 

// let asianCapitals = ['Seoul', 'Dhaka', 'Manilla', 'Bangkok']
// let result = asianCapitals.find()

// console.log(result)

Longhand vs Shorthand Syntax

The code syntax we just practiced is considered longhand; you're more likely to see and use arrow functions with array iterators

Basic Arrow Function Syntax

Arrow functions allow us to write iteration methods a bit more concisely

  • Remove the function keyword
  • Add an arrow before the first opening curly brace {
  • That's it!
// ......
let result = asianCapitals.find( (city) => {
  if(city === 'Manilla'){
    return city
  }
} )
// ......

Basic Arrow Function Syntax

We can strip the code down even more!

  • Because we only have one parameter: 'city', in the testing function, we can omit the () around it 
// ......
let result = asianCapitals.find( city => {
  if(city === 'Manilla'){
    return city
  }
} )
// ......

Arrow Function Shorthand

The rules we learned for writing arrow function shorthand also apply with iterators

If an arrow function expression is simple enough, we can omit the return keyword and keep our code to one line 

// ......

let result = asianCapitals.find( city => city === 'Manilla' )

// ......

How to Use Array Iteration Methods in JavaScript

.map()

Creates a new array that contains the results of an action that's been applied to every element in the original array

Array.prototype.map() - MDN

.map()

Square each item in the numbers array and return the results in a new array 

const numbers = [1,2,3,4]

const doubleNumbers = numbers.map( num => {

  return num * num

})

console.log(doubleNumbers) // Output: [1,4,9,16]

// Arrow syntax shorthand version:

// numbers.map(num => num * num)

Array Index Values

When looping through items in an array, iterators are also keeping track of index positions

We can reference or return these index positions as results 

const zeeNames = ['Zoe', 'Zadie', 'Zazee']

let getNamePosition = zeeNames.map((name, index) => {
  return `The name ${name} has an index value of ${index} in the array.`;
})

console.log(getNamePosition)

.filter()

Creates a new array with elements that pass the test condition of the parameter function

Array.prototype.filter() - MDN

.filter()

Filter out the words that contain the string 'cat' into a new array 

let catStrings = ['avocado', 'advocate', 'catty', 'catacomb']

let nonCatStrings = catStrings.filter( item => {

  return item.includes('cat')

})

console.log(nonCatStrings) 

// Arrow syntax shorthand version:

// catStrings.filter(item => !item.includes('cat'))

Return Values

While some iteration methods create a new array, others return elements from the original array or a boolean value; and one particular method, the forEach, does not return any value at all

.forEach()

Runs an action on every element in an array; unlike other methods, it does not return an actual value 

let ageList = [4,5,6,7]

const updatedAgeList = ageList.forEach( element => element + 5 ) 

console.log(updatedAgeList) // Returns undefined

.forEach() - MDN

.forEach()

let ageList = [4,5,6,7]

ageList.forEach( element => console.log(element + 5) )

.forEach() - MDN

.forEach()

The forEach behaves similarly to the for loop and the for of loop 

let ageList = [4,5,6,7]

ageList.forEach( element => console.log(element + 5) ) 
// Output: 9, 10, 11, 12

for(let i = 0; i  < ageList.length; i++){
  console.log( ageList[i] + 5)
}
// Output: 9, 10, 11, 12

for(let age of ageList){
  console.log(age + 5)
}
// Output: 9, 10, 11, 12

When to Use .forEach?

forEach can be an alternative to using a for loop or a for...of loop; all three iterate through every element in an array

What About Other Methods?

A few tips:

  • Use .map to change items in an array
  • Use .filter to select a subset of an array
  • Use .find to select one item in an array
