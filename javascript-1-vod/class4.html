Conditional Chaining:
Switch Statement

A switch statement is an alternative to writing
long chains of elseif condition blocks

Switch Statement (MDN)

Switch Statement Syntax

switch (expression) {
  case value1:
    // do this if the expression (condition) matches value1
    break; // then exit
  case value2:
    // do this if the expression (condition) matches value2
    break; // then exit
  case value3:
    // do this if the expression (condition) matches value3
    break; // then exit
    ...
  default:
    // do this when nothing matches the condtion
}
Switch Statement (MDN)

Switch Statement Syntax

  • switch keyword initializes the conditional
  • expression is the condition to compare against
  • case valueN:
    • The case clause defines what has to be compared to the expression and what to do if there's a match
    • valueN is the value to compare against the expression
  • break is an optional keyword that ends the program if there's a match found
  • default sets the instruction to run if no match is found

Switch Statement

Using a switch statement on the
day-of-the-week conditional exercise from Class 3

let today = 'thursday';

switch (today){
  case 'monday':
    console.log( 'Here we go again....' )
    break;
  case 'tuesday':
    console.log( 'Are we done yet?' )
    break;
  case 'wednesday':
  case 'thursday':
    console.log( 'Getting a bit excited...' )
    break;
  default:
    console.log( 'I love the weekends!' )
}

Edge Case: Logical Operators

  • Caveat: When using logical operators in a switch
    statement, the expression to evaluate against
    should be a boolean value (true or false)
  • All case clauses have to be logical expressions

Functions

  • A function is a block of code with instructions to perform a particular task
  • Functions are reusable--they can be called multiple times to perform the same task in different parts of a codebase
  • Functions make our code organized and more efficient
JavaScript Functions (W3Schools)
Functions (MDN)

Function Declaration

We declare a function with the keyword function, a function name, a pair of parentheses and curly braces

// The syntax of a function declaration:

function funtionName(){
  // instructions for the function
  // to carry out 
}

Function Call

But declaring a function is not enough. It then needs to be invoked or called or run. These are all ways of saying we want to use or execute the function.

// The syntax of a function call:

functionName()

Sample Function

// Declaring the function
function parrotFacts(){

  // What the function should do
  console.log('Parrots are very intelligent birds.');

} 

// Calling the function
parrotFacts()

Function Parameters

  • Functions can take (optional) input values known as parameters
  • A parameter is a placeholder for a value that the function will use when called
// Syntax of using or passing a parameter:

function friendlyGreeting( anyName ){

  console.log(`Hi ${anyName}`)

}
  
friendlyGreeting(...)

Function Argument

  • An argument is the actual value passed to the function to use (when invoked) instead of the placeholder
  • Arguments can be any data type: Strings, Numbers, Boolean, Arrays, etc.
// Syntax of using an argument:

function friendlyGreeting( anyName ){

  console.log(`Hi ${anyName}`)

}
    
friendlyGreeting('Tamara')
// We've passed an argument, the string 
// 'Tamara' to the function to use

Function Argument

  • We can now reuse the function multiple times with different arguments 
// Syntax of using an argument:

  function friendlyGreeting( anyName ){
  
    console.log(`Hi ${anyName}`)
  
  }
    
  friendlyGreeting('Tamara')
  friendlyGreeting('Jamal')
  friendlyGreeting('Cindy')
