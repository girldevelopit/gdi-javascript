Numbers

Integers and decimals make up the number data type

let age = 35

let taxRate = 27.5

Math Operators

We can use standard math operators with the number type

let sum = 35 + 20 // Addition

let difference = 200 - 130 // Subtraction

let product = 10 * 50 // Multiplication

let square = 3 ** 3 // Exponentials

let dividend = 18 / 3 // Division

let remainder = 18 % 4 // Modulo

/* Modulo is a special operator that returns
the remainder of a division */
Note: Calculations in JavaScript follow the PEMDAS rule.

Math Operators & PEMDAS

Calculations in JavaScript follow the PEMDAS or Order of Operations rule.

Quick practice: First, calculate the math expression below without using the console. Then confirm the result in the console.

let total = 4 * (2 + 9) - 3

Math Operators: Variables

We can also perform math using number variables

let currentYear = 2022

let birthYear = 1993

let age = currentYear - birthYear

console.log(age)

Math Operators: Incrementing

The increment operator ++ adds one to a value

let age = 2

age++;

console.log(age) // This should be 3

let newAge = ++age;

console.log(newAge) // This should be 4

Math Operators: Decrementing

The decrement operator -- subtracts one from a value

let year = 1992

year--;

console.log(year) // This should be 3

let oldYear = --year;

console.log(oldYear) // This should be 4

Math Operators: Compound Assignments

  • Compound assignment operators combine an assignment and a math operator
  • They take the value to the right of the assignment operator and apply it to the variable
let tip = 15

console.log( tip += 20 ) // 35

// This is the same as:
// let tip = 15
// tip = tip + 20

Math Operators: Compound Assignments

The main math operators can be turned into compound operators

let tip = 15

console.log( tip += 20 ) // 35

console.log( tip -= 10 ) // 25

console.log( tip *= 4 ) // 100

console.log( tip /= 8 ) // 12.5

Strings and Compound Operators

  • Strings can be combined using the += operator
  • Using other operators with strings will return NaN
let greeting = "hello"

greeting += " world"

console.log(greeting) // hello world

greeting -= " world"

console.log(greeting) // NaN

NaN

NaN stands for Not-A-Number, which is an actual number value!

Typically encountered when an operation or calculation has failed

An Essential Guide to JavaScript NaN (javascripttutorial.net)

Combining Strings and Numbers

  • Strings and numbers can be combined with + operator
console.log( 7 + " oranges" ) // 7 oranges

let myAge = "I am "

console.log(myAge += 8) // I am 8
  • This is not recommended; turn the number into a string data type instead

Writing Numbers as Strings

  • Numbers can be written as strings by adding quote marks
  • But calculations done this way are unreliable and can introduce bugs
let num1 = '5' // Number as a String

console.log (num1 + 5) // 55

console.log (num1 * 5) // 25

console.log (num1 - 5) // 0

Checking Data Types

typeof Operator

To make sure we're working with the correct data types, use the typeof operator for verification; this can prevent bugs

let animal = 'dog'
console.log(typeof animal) // string

console.log(typeof 7) // number
console.log(typeof '7') // string

console.log(true) // boolean

Boolean

The boolean data type has two values true and false

let catsAreTheBest = true;

let dogsAdoreCats = false;

Default Boolean Values

All variables in JavaScript have a default Boolean value:

They're either "truthy" or "falsy"

Truthy and Falsy Boolean Values

    Truthy
  • true
  • 4
  • 'hello'
  • '0'
  • []
  • {}
    Falsy
  • false
  • 0
  • -0
  • ' '
  • null
  • undefined

Boolean Function

We can use the Boolean function to determine
if a value is truthy or falsy

Boolean('Jackie') // true

Boolean (-0) // false

Comparison Operators

Comparison operators compare two values
and return a boolean (true or false)

Operator Name
a === b Identical
a !== b Not identical
a < b Less than
a > b Greater than
a <= b Less than or equal to
a >= b Greater than or equal to

Comparison Operators

  • > greater than
  • < less than
  • >= greater than or equal to
  • <= less than or equal to
console.log( 19 > 20 ) // false

console.log( 5 >= 5 ) // true

console.log( 0 < 2 ) // true

console.log( 6 <= 8 ) // true

Equality Operators

Equality operators test if two values are the same. The equality operators look very similar but work differently.

Loose Equality

Double equals '==' or 'loose equality'
evaluates if values are the same

console.log( 5 == '5' ) // true
JavaScript — Double Equals vs. Triple Equals (Codeburst)

Strict Equality

Triple equals '===' or 'strict equality'
evaluates value AND type

console.log( 5 === '5' ) // false

// 5 as a number is not the same as 5 as a string value

To prevent bugs, use strict equality over loose equality

JavaScript — Double Equals vs. Triple Equals (Codeburst)

Inequality

Both equality operators have their negation counterparts:

  • Loose inequality
console.log( 5 != '5' ) // false
  • Strict inequality
console.log( 5 !== '5' ) // true

Conditionals

Conditional statements control the behavior of our application. They determine the conditions under which code should run, or in what way it should be executed.

if Statement

If a condition meets a criteria, then run the code

if(condition is true){

  then do something...
}

let catsSay = 'meow';

if(catsSay === 'meow'){

  console.log(`I'm a cat and I love to sleep all day`)
}

if-else Statement

if-else extends the if statement by taking into account what should happen if the original condition is NOT met

if(condition is true){

  do b...
}

  console.log(`I'm a cat and I love to sleep all day`)

} else {
  
  console.log(`I must be a dog then.... Woof!`)

}

Logical Operators

Logical operators check the logic between values or
conditions. There are three operators:

  • AND &&
  • OR ||
  • NOT !

JavaScript Comparison and Logical Operators (Programiz)

&& : The AND Operator

The AND operator returns true if two conditions being evaluated are both true; otherwise it returns false

let sandyAge = 7

let jamalAge = 5

console.log( sandyAge > 4 && jamalAge < 9 ) // true

console.log( sandyAge > 4 && jamalAge > 9 ) // false

|| : The OR Operator

The OR operator returns true if one condition is true;
it returns false when both are false

let sandyAge = 7

let jamalAge = 5

console.log( sandyAge > 4 || jamalAge > 9 ) // true

console.log( sandyAge < 4 || jamalAge < 9 ) // false

! : The NOT Operator

The NOT operator returns the opposite condition of what's evaluated. If a value is true, the operator will return false and vice versa

let sandyAge = 7

let jamalAge = 5

console.log( !sandyAge ) // false

console.log( !jamalAge > 3 ) // false

!(true) // false

!(false) // true
