for...in Loop

Looping Objects

The for...in loop iterates over the properties of an object

It has the same syntax or setup as the for...of

for( key in object ){

  // code to run

}

JavaScript For In (w3schools)

for...in Loop

  • for...in iterates and returns an object's keys
  • Each key is used to access its respective value
let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

for( key in carObj ){

  console.log( carObj[key] )

}
// Output: BMW, 2020, red

for...in VS for...of

  • While the for...in loop can iterate over an array, order is not guaranteed
  • Use the for...of for iterable (countable) data such as arrays and strings
for( key of carObj ){

  console.log( carObj[key] )

}
// Output: Error!
// carObj is not iterable

Object Methods (MDN)

Object.keys()

Returns an array of an object's keys

let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

console.log( Object.keys(carObj) )

// Output: ['model', 'year', 'color']

Object.keys() - MDN

Object.values()

Returns an array of an object's values

let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

console.log( Object.values(carObj) )

// Output: ['BMW', '2020', 'red']

Object.values() - MDN

Object.entries()

Returns arrays of an object's key:value pairs

let carObj = {
  model: 'BMW',
  year: '2020',
  color: 'red'
}

console.log( Object.entries(carObj) )

// Output: 3 arrays

Object.seal() - MDN

.seal()

With Object.seal(), new properties cannot be added to an object, nor can existing ones be removed; but existing properties can be updated

Object.isSealed() method checks if an object is sealed; it returns a boolean

.freeze()

With Object.freeze(), new properties cannot be added to an object, existing ones cannot be removed nor updated; it is frozen!

Object.isfrozen() method checks if an object is frozen; it returns a boolean

Immutable Objects

A frozen or sealed object cannot be reversed; there is no "unseal" or "unfreeze" method

But using the Object.assign() method, our immutable object can be copied or cloned to a new object that is mutable

Classes

Who doesn't love a good muffin?!

But imagine that any time you wanted a single muffin, you would have to bake it from scratch!

Wouldn't it be easier to have a muffin template?

You could create lots of muffins!

You could even create a variety of muffins!

JavaScript Classes are like muffin templates--but for objects!

Classes

Classes are templates for creating objects

When we create one class, we can then create as many objects as we want from that class template

JavaScript Classes (w3schools)

Class Syntax

  • We declare a class using the class keyword
  • We give the class a name
  • Class names are always capitalized
class Muffin {

}

Class Syntax

  • Each class declaration requires a constructor function
  • The constructor sets the properties of the class
  • The constructor can take multiple parameters
  • Every object created from the class inherits or starts off with these properties
class Muffin {
  constructor ( name, flour, eggs, sugar ){
    this.name = name
    this.flour = flour,
    this.eggs = eggs,
    this.sugar = sugar
  }
}

Class Syntax

Functions (methods) can also be included in a Class

class Muffin {
  constructor ( name, flour, eggs, sugar){
    this.name = name
    this.flour = flour,
    this.eggs = eggs,
    this.sugar = sugar
  }

  tasteTest(){
    console.log(`This is a great ${this.name} muffin!`)
  }
}

Class Syntax

We've created a basic Muffin class or template, now let's create an object from it!

Creating an Object from a Class

  • Declare and name a new variable
  • Use the new keyword followed by the Class name and a pair of ()
  • In between the parentheses, add the properties that correlate to the parameters set by the constructor
const blueberryMuffin = new Muffin( 
  'blueberry', 'white', 4, '1 cup' 
)

console.log(blueberryMuffin)

Empty Arguments?

When creating an object from a class, if we skip a constructor parameter, the object returns undefined for that key

Example: Let's omit the value for the sugar property when using the Muffin class

const raspberryMuffin = new Muffin( 
  'raspberry', 'almond', 4 
)

console.log(raspberryMuffin)
// Output:
// {name: 'raspberry', flour: 'almond', eggs: 4, sugar: undefined}

Default Arguments

To avoid this, we can add default arguments when creating a Class

Default Arguments

A default argument is used as a default value in the event an expected value is missing or omitted

class Muffin {
  constructor ( name, flour, eggs, sugar = '1 cup' ){
    this.name = name
    this.flour = flour,
    this.eggs = eggs,
    this.sugar = sugar,
  }
}

let cornMuffin = new Muffin ('corn muffin', 'cornmeal', 2)

console.log(cornMuffin)
// Output:
// {name: 'corn muffin', flour: 'cornmeal', eggs: 2, sugar: '1 cup'}

What if we wanted to modify our Muffin class so we can also create Vegan muffins?

Inheritance or Creating subclasses

We can create a subclass that inherits all the properties of the parent class, and can then be modified with new functionality

extends Keyword

The extends keyword creates a class that is a child class or subclass of another class

class VeganMuffin extends Muffin {
  constructor (name, flour, eggs, sugar, liquid){
    super(name, flour, eggs, sugar)
    this.liquid = liquid
  }

  veganDisclaimer(){
    console.log('No animal products used')
  }
}

super Keyword

When extending a class, it is required to use the super keyword to reference the parent class' properties

super Keyword

In the example below, we've added a new parameter 'liquid' to the VeganMuffin subclass

We use super to reference the original properties from the parent Muffin class

We then only need to initialize or use liquid

class VeganMuffin extends Muffin {
  constructor (name, flour, eggs, sugar, liquid){
    super(name, flour, eggs, sugar)
    this.liquid = liquid
  }

  veganDisclaimer(){
    console.log('No animal products used')
  }
}

Calling Parent and Child Methods

The new object can use methods from its own class and the parent class

 // Method from its own class

// No animal products used!

// Method from parent class

// This is a great vegan corn muffin!
